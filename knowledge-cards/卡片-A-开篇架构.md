# 卡片-A：开篇架构

## 📋 知识卡片列表

### 卡片 A-1：黄金三章六要素

| 字段 | 内容 |
|------|------|
| **关键词** | 开篇、黄金三章、六要素 |
| **定义** | 开篇必须让读者/编辑清晰感知六个要素，缺一即不成立。 |
| **规则** | 1. 舞台搭建：开篇即给出世界信息（"大奉京兆府监牢"）<br>2. 事件起因/今生延展性：原主遗留的驱动事件，非简单背景<br>3. 合理逻辑目标：主角结合现状产生的驱动力<br>4. 明确变量：穿越者身份+金手指<br>5. 解题行进：朝目标推进的具体剧情<br>6. 制造落差：有起伏才有后续爽点爆发 |
| **案例** | 《大奉打更人》：睁眼穿入监牢（舞台）→ 原主失职待斩（起因）→ "必须破案自救"（目标）→ 穿越者+现代化学知识（变量）→ 验尸查案（解题）→ 从死囚到立功（落差） |
| **常见错误** | 1. 开篇只有世界观设定堆叠，没有事件起因和驱动目标<br>2. 穿越后主角被动等剧情推，没有"今生延展性"提供的主动方向 |
| **关联卡片** | A-2, A-7, G-3 |

### 卡片 A-2：睁眼穿标准架构

| 字段 | 内容 |
|------|------|
| **关键词** | 睁眼穿、穿越开局、叙事节奏 |
| **定义** | 穿越文最经典的开局方式：主角睁眼即已身处异世界，直接切入正在进行的情境。 |
| **规则** | 1. 开局直白切：两句话交代时间/地点/天气 + 主角正在什么情境<br>2. 前世今生衔接：前世有用写前世，没用直接写今生；利用前世最后死亡带来今生引导<br>3. 今生延展性（核心）：不是身份背景，而是原主遗留的驱动事件，为后续行为提供明确方向<br>4. 前世今生信息延展性：清晰的叙事、明确的目标 |
| **案例** | 案例A："他睁开眼，头顶是发霉的茅草屋顶。原主的记忆涌入——三天前，他把全家三年的积蓄拿去赌坊，输得精光。"（延展性：全家期望压在主角肩上）<br>案例B：前世战死沙场→今生睁眼发现自己成了逃兵，随时被军法处置（前世死亡引导今生危机） |
| **常见错误** | 1. 睁眼后花几千字描写原主生平，没有延展性事件驱动<br>2. 前世写一大堆与今生无关的记忆，信息浪费 |
| **关联卡片** | A-1, A-8, C-10 |

### 卡片 A-3：分总式穿越确认

| 字段 | 内容 |
|------|------|
| **关键词** | 穿越确认、分总式、认知过渡 |
| **定义** | 主角穿越后的确认过程采用"分→总"结构，逐步从局部感知到整体认知。 |
| **规则** | 1. 先写主角感知到的局部异常（陌生的床/衣服/身体）<br>2. 再整合记忆碎片，确认穿越事实<br>3. 最后给出主角的主观反馈和目标确立<br>4. 整个控制在1-2章内完成，不拖沓 |
| **案例** | 主角先摸到身下是硬木板床（异常1）→ 看到铜镜里陌生的脸（异常2）→ 涌入原主记忆（确认）→ "既然占了你的身子，你未了的事我来办"（反馈+目标） |
| **常见错误** | 1. 穿越确认花了三章以上，节奏断裂<br>2. 主角确认穿越后没有主观反馈，像旁观者而非参与者 |
| **关联卡片** | A-2, A-6 |

### 卡片 A-4：对话开局双要求

| 字段 | 内容 |
|------|------|
| **关键词** | 对话开局、普适性、信息密度 |
| **定义** | 选择对话开局时，必须同时满足普适性和信息密度两项要求。 |
| **规则** | 1. 普适性：对话中涉及的概念读者必须能理解（"副统领""银钱"有通用认知基底）<br>2. 信息密度：对话必须伴随快节奏事件推进，是前三章最核心的信息载体<br>3. 对话不能是寒暄废话，每一句都要出设定或推进<br>4. 对话后紧跟主角主观反馈 |
| **案例** | "阿胜，拉王哥一把，借些银钱，周旋。等哥哥买通吴统领，当上副统领，第一个提拔你。"——一句话交代了人物关系（阿胜/王哥）、当前困境（借银钱周旋）、未来目标（当副统领）、权力结构（吴统领） |
| **常见错误** | 1. 开篇对话大量使用自创术语，读者根本不知在说什么<br>2. "你好""吃了吗"式无效对话，零信息密度 |
| **关联卡片** | A-7, C-14, C-15 |

### 卡片 A-5：环境描写倾向性

| 字段 | 内容 |
|------|------|
| **关键词** | 环境描写、倾向性、正衬反衬 |
| **定义** | 环境描写必须有倾向性——要么不做形容词，做了就必须与后续风格契合。 |
| **规则** | 1. 要么不做形容词（如干巴巴的"大奉京兆府监牢"），做了就要有目的<br>2. 正衬：环境烘托当下氛围（冷风衬托灵堂阴森）<br>3. 反衬：环境反差制造转折感（暖风闲适 → 陡然急转直下）<br>4. 文字描写用具体可感元素：红嫁衣/白脸红唇/脚趾涂红指甲油；空泛形容词无画面感 |
| **案例** | 正衬："阴风从破窗灌入，纸钱打着旋落在棺木上。"<br>反衬："三月的暖阳照在青石板上，巷子口的桃花正盛。可拐过这道弯，血腥味扑鼻而来。" |
| **常见错误** | 1. 开篇花大段描写风景，和剧情毫无关联<br>2. 用了"阴森恐怖""美丽动人"等空洞形容词，读者脑中无具体画面 |
| **关联卡片** | A-7, C-7, H-11 |

### 卡片 A-6：设定穿插规则

| 字段 | 内容 |
|------|------|
| **关键词** | 设定穿插、走哪写到哪、一事不烦二主 |
| **定义** | 世界观设定不能一次性堆叠，必须分拆融入叙事流程中。 |
| **规则** | 1. 从轮廓到清晰：先给大世界观，再收束到主角个人<br>2. 走哪写到哪：主角看到什么写什么，有次序补充设定<br>3. 一事不烦二主：开篇出现的角色/设定，后续必须有明确用途<br>4. 穿插后必须转回"正在进行时"：不能用"与此同时""三年后"等生硬切走 |
| **案例** | 先写"大奉京城，东市西市"（大轮廓）→ 主角走在青石板路上看到两旁店铺（中景）→ 进了一家药铺抓药（近景）→ 掌柜说"这味朱砂最近涨了五成"（具体困境/设定）→ 转回主角内心："得想办法搞钱。"（回归主线） |
| **常见错误** | 1. 开篇三千字世界观设定，像明书<br>2. 写了一个配角/势力然后就消失了，违反"一事不烦二主" |
| **关联卡片** | A-1, G-5, G-6 |

### 卡片 A-7：信息密度密集型开篇

| 字段 | 内容 |
|------|------|
| **关键词** | 信息密度、一箭多雕、每一句话都不是随意 |
| **定义** | 开篇每一句话都承担多重功能——设设定、塑人设、推剧情。 |
| **规则** | 1. 人物对话和行为必须一箭多雕<br>2. 开篇前三章信息密度最高，是全书定调关键<br>3. 每一句话都不是随意而为——要么带设定、要么刻人设、要么推剧情<br>4. 删除零功能的装饰性文字 |
| **案例** | "刘管家站在院门外，手里捏着账本，眼皮都不抬：'三少爷，这个月的例银，府里紧，您看……'"——同时完成：①人物身份（刘管家）②人物态度（不敬：眼皮不抬）③当前危机（例银没了）④权力关系（管家>少爷）⑤推动主角必须解决经济问题 |
| **常见错误** | 1. 大段环境/心理独白，对推动剧情零贡献<br>2. 配角出场只写了外貌，没有给出任何信息或推动 |
| **关联卡片** | A-4, A-5, C-12 |

### 卡片 A-8：前世今生衔接

| 字段 | 内容 |
|------|------|
| **关键词** | 前世今生、记忆融合、死亡引导 |
| **定义** | 穿越文中前世记忆与今生情境的衔接方式，决定开篇的流畅度和动力感。 |
| **规则** | 1. 前世有用写前世，没用直接写今生（有取舍）<br>2. 最佳衔接方式：利用前世最后的死亡/遗憾带来今生引导<br>3. 衔接要自然流畅，不生拐硬套<br>4. 前世技能/知识必须能在今生发挥作用（信息差来源） |
| **案例** | 前世化学家死于实验室爆炸 → 今生睁眼发现自己穿越到古代炼毒世家 → 前世知识立刻成为金手指<br>失败案例：前世写了一千字的大学校园生活，和今生穿越修仙世界毫无关系 |
| **常见错误** | 1. 前世今生各写各的，中间没有因果或情感桥梁<br>2. 前世信息完全无用，纯水 |
| **关联卡片** | A-2, B-11, C-10 |

### 卡片 A-9：主角主观反馈（90%篇幅）

| 字段 | 内容 |
|------|------|
| **关键词** | 主观反馈、驱动力、主角弧光 |
| **定义** | 紧跟在场景设定穿插之后，主角必须给出主观反馈，这是网文90%篇幅必须遵循的流程。 |
| **规则** | 1. 对剧情信息点进行明确总结（主角心里怎么想）<br>2. 体现主角弧光（三观、性情通过反馈展现）<br>3. 明确目的性和驱动（让读者一眼看到他要干什么）<br>4. "主角内心反馈是写一本网文的重中之重" |
| **案例** | 设定穿插后："练成武功后，什么女人找不到。"——读者一眼看到主角驱动力（练武→变强→获得女人），直白有效。<br>好的反馈："这具身体的原主是个废物，但面板告诉我，只要练就能涨。那就练。" |
| **常见错误** | 1. 写了场景和事件，但主角没有内心反馈，像个木偶<br>2. 反馈空洞："我一定会努力的！"——没有具体目标和驱动力 |
| **关联卡片** | F-1, B-11, C-1 |

### 卡片 A-10：四种基本开篇节点

| 字段 | 内容 |
|------|------|
| **关键词** | 开篇节点、切入时机、剧情起点 |
| **定义** | 网文开篇有四种常见切入节点（时间点），选择哪个决定了叙事节奏和读者代入。 |
| **规则** | 1. 危机节点：主角面临迫在眉睫的危险（最紧凑）<br>2. 转折节点：主角命运发生转折的时刻（穿越/觉醒/退婚）<br>3. 日常节点：从日常切入，快速拉入异常（节奏慢但自然）<br>4. 高潮前置节点：先写一个精彩场景做钩子，再倒叙解释（吸引力最强） |
| **案例** | 危机节点：开篇主角已被绑在刑场<br>转折节点：开篇主角刚刚穿越或刚觉醒金手指<br>高潮前置："那一剑，他等了三年。"→ 倒叙解释三年前发生了什么 |
| **常见错误** | 1. 选了日常节点但节奏太慢，三章还在吃早饭<br>2. 高潮前置和正文风格不统一，读者觉得被骗 |
| **关联卡片** | A-2, J-9, C-1 |

### 卡片 A-11：书名漏斗效应

| 字段 | 内容 |
|------|------|
| **关键词** | 书名、漏斗效应、信息预判 |
| **定义** | 书名是读者接触作品的第一层漏斗，决定读者带着什么预期点开你的书。 |
| **规则** | 1. 书名必须传达金手指/题材/核心卖点至少一个<br>2. 书名越精准，点击进来的读者越匹配，留存率越高<br>3. 书名和简介、正文必须风格一致，否则产生认知断裂<br>4. 好书名让读者在点进来之前就知道"这是本什么书" |
| **案例** | 《我的面板能无限升级》——读者一看就知道是面板流/升级流<br>《从xxx开始xxx》——经典穿越/系统流书名模板，自带预期 |
| **常见错误** | 1. 书名文艺但不知所云，读者不知道是什么类型<br>2. 书名说是系统流，正文完全是传统修仙，读者流失 |
| **关联卡片** | L-1, L-3, D-1 |

### 卡片 A-12：简介反常噱头法

| 字段 | 内容 |
|------|------|
| **关键词** | 简介、反常噱头、颠覆预期 |
| **定义** | 简介的一种写法——用反常/颠覆预期的事件或设定作为钩子，制造好奇。 |
| **规则** | 1. 找出与读者常识/预期违背的设定或事件<br>2. 用这个"反常点"做开局钩子<br>3. 反常后必须给出合理的逻辑解释（否则是故弄玄虚）<br>4. 噱头之后必须连接全书脉络，让读者知道后续看什么 |
| **案例** | "穿越第一天，我被宗门除名了。第二天，师尊跪在我门口求我回去。"（反常：为什么除名了反而被求？）→ 连接金手指说明原因 |
| **常见错误** | 1. 噱头反常但和后文毫无关系，纯粹的"标题党"<br>2. 反常没有合理解释，读者看完简介觉得作者在扯淡 |
| **关联卡片** | L-1, L-3, L-6 |
