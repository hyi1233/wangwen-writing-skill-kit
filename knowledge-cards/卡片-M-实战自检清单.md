# 卡片-M：实战自检清单

## 📋 说明

本卡片整合已有文档的完整自检清单，并补充新增检查项（来源于新增知识点）。

---

## 已有自检清单（来自原文档）

### □ 开篇六要素检查
- [ ] 舞台搭建：读者知道这是什么世界
- [ ] 事件起因/今生延展性：主角目标与事件绑定
- [ ] 合理逻辑目标：困境的解决方法
- [ ] 明确变量：穿越者+金手指
- [ ] 解题行进：剧情推进方向明确
- [ ] 制造落差：有起伏才有爽点爆发

### □ 信息穿插检查
- [ ] 没有设定堆叠（掰开揉碎，见缝插针）
- [ ] 所有写出的信息前3-4章有用
- [ ] 穿插后转回正在进行时（没用"与此同时"等生硬转场）
- [ ] 一事不烦二主（写到的必有后续）

### □ 爽感设计检查
- [ ] 困境 + 希望（两者成对出现）
- [ ] 有明确参照物/打脸对象
- [ ] 信息差：读者知道主角能赢
- [ ] 递进式情绪下压（不是一下压到底）
- [ ] 主角有解决手段/翻盘信心
- [ ] 收获 > 付出

### □ 人物塑造检查
- [ ] 主角人设用行为展现（不是定义宣告）
- [ ] 配角脸谱化完成快（一两句话）
- [ ] 有主观视角黄金流程（感知→反馈→动机→驱动）
- [ ] 高光时有围观视角衬托

### □ 文字质量检查
- [ ] 没有解释尾巴（动作后没有"这表现了……"）
- [ ] 情感有附着物（不是空洞抒情）
- [ ] 人物有缺点/味道（不是完美无缺）
- [ ] 动词精准有力（不是形容词堆砌）
- [ ] 生活化比喻（不是文学化比喻）
- [ ] 多感官细节（不只写"看到的"）

---

## 🆕 新增检查项（按类别补充）

### □ 金手指设计检查
- [ ] 金手指与主线剧情有明确关联（不是装饰品）
- [ ] 核心机制没有不断添加新功能（一个机制用到死）
- [ ] 金手指不是主角，主角行为由主观能动性驱动
- [ ] 信息差/期待感有效维持（读者知道主角能成，配角不知道）
- [ ] 金手指有明确的代价/限制（代偿机制）
- [ ] 金手指效果数值化，读者能一眼看到进度
- [ ] 没有反套路金手指时"立锚点→亲手打碎"的设计感

---

### □ 叙事技巧检查
- [ ] 核心事件用倒果为因方式设计（先定落点再铺路）
- [ ] 信息差至少一种模式有效运作
- [ ] 细节描写使用信息锚点（具体可感，不是空泛形容词）
- [ ] 倾向性描写到位（暗示不说明，让读者自推）
- [ ] 至少一处"一笔三用"信息效率细节
- [ ] 路人闲聊承担功能性信息发射（不是纯闲聊）
- [ ] 关键信息有"不写但谁都知道"的留白处理
- [ ] 没有"与此同时""三年后"等生硬转场

---

### □ 情绪管理检查
- [ ] 困境和希望成对出现，间隔不超过1-2章
- [ ] 有明确的爽点链条（困境→信息差→压迫→行动→反转→收获）
- [ ] 反向Flag有效设置（配角断言+读者窃喜+必然回收）
- [ ] 递进式情绪下压有多步阶梯（不是一步压到底）
- [ ] 有拉扯感制造（爽点前适度延迟）
- [ ] 存在核心意难平（全书情感锚点）
- [ ] 时间错位感运用（未来映照现在/过去暗示未来）

---

### □ 爽感设计检查（补充）
- [ ] 有具体参照物对比，不是泛泛对比
- [ ] 事件强度递进（不是用弱对手显主角强）
- [ ] "我越弱我越强"维度升级有提前铺垫
- [ ] 反差-反转有前文伏笔支持（不是机械降神）
- [ ] 选择了"弹幕最多的打法"（读者最想看的方向）
- [ ] 欲扬先抑有真实感（不是假打压）

---

### □ 人物塑造检查（补充）
- [ ] 角色语言有差异化DNA（不看标签知道谁在说话）
- [ ] 女主不白给不倒贴（有独立目标和行动线）
- [ ] 人物出场公式完整（标签动作+关系定位+闲笔）
- [ ] 有异常正常化特质（主角自带异于常人的点）
- [ ] 美女描写用文字锚点不用空泛形容词
- [ ] 一场戏收尾有差异化反应（不是一种态度）

---

### □ 伏笔设计检查
- [ ] 短/中/长三层伏笔同时存在
- [ ] 有契诃夫之枪（特意描写的细节后续有用）
- [ ] 反向伏笔有误导性+反转+前文暗线
- [ ] 葡萄串式伏笔串联至少2-3个环节
- [ ] 短线事件同时推进了主线伏笔
- [ ] 结论锚点的权威已建立在前文

---

### □ 去AI化检查（补充）
- [ ] 零"像"字：行文中没有"像/仿佛/宛如/好似"的类比
- [ ] 动作后没有解释尾巴（写完立刻停笔）
- [ ] 情感有生理反应替代心理描写
- [ ] 情感物化到具体物件上（思念=饺子等）
- [ ] 没有上帝旁白（"他是一个好人""这世界很残酷"）
- [ ] AI高频词禁区清零（璀璨/熠熠生辉/心中涌起/嘴角勾起）
- [ ] 对话有潜台词（表面意思≠真实意思）
- [ ] 角色手里不空（有道具/动作绑定）
- [ ] 角色有不完美之处（真缺点/异味/尴尬）

---

### □ 文字质量检查（补充）
- [ ] 删除了描写性"是"字句，替换为动作
- [ ] 动词精准，没有"副词+弱动词"组合
- [ ] 三层语言系统区分（叙述层/对话层/内心独白层）
- [ ] 有短句节奏交替（短-短-长-短）
- [ ] 重要动作/细节独立成段（手术刀式分段）
- [ ] 恐怖/反转最后一句是短句定格（≤10字）
- [ ] 恐怖场景有物理递进（听觉→触觉→视觉）

---

### □ 节奏控制检查
- [ ] 节奏与剧情排列匹配（不是只看字数）
- [ ] 每1-2章有收获-期待-演绎循环
- [ ] 章末有钩子（卡期待/卡转折/卡危机）
- [ ] 平静与爆发交替出现（没有连超5章同节奏）
- [ ] 黄金三章结尾有明确的章末钩
- [ ] 情绪标注细纲曲线有规律波动

---

### □ 简介检查
- [ ] 三要素齐全（困境+金手指+全书脉络）
- [ ] 简介风格与书名、正文一致
- [ ] 开局钩子有反常/颠覆预期的要素
- [ ] 简介长度适中（不超过200字）

---

## 使用建议

1. **写完一章**：检查"文字质量"和"去AI化"两个区块
2. **写完一个事件团**：检查"爽感设计""情绪管理""伏笔设计"区块
3. **完稿后通查**：全部检查项过一遍
4. **新开书时**：重点检查"金手指设计"和"开篇"区块
5. 不需要每条都打勾——但看到没打勾的要明确"为什么没做"
