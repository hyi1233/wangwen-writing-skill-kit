# 网文金手指设计

> 核心信条：金手指是工具，主角才是驱动力。一个机制用到死。

## 触发条件

用户提到以下任一场景时自动激活：
- 设计金手指 / 写系统 / 写面板 / 设计能力
- 修改金手指机制 / 金手指与剧情绑定

## 核心规则

1. **金手指是工具不是主角** — 金手指提供"可能"，主角决定"做什么"。90% 篇幅必须是主角主观反馈
2. **一个机制用到死** — 选定一个核心机制后不再添加，围绕它做无限演绎
3. **金手指必须有作用** — 每个事件都问"和金手指有什么关系？"无关可删
4. **代价驱动剧情** — 每次使用有明确代价，代价需要解决 → 推动新事件
5. **数据化降低门槛** — 简单直观的数值让读者一眼看到成长

## 工作流

### Step 1：选类型

从五大赛道中选一种（上限从高到低）：

| 类型 | 核心看点 | 关键操作 |
|------|---------|---------|
| **词条海克斯** | 文字游戏+多变组合 | 表面释义做有趣演绎，机制多变但奖励实打实（最近最热） |
| **错位金手指** | 认知偏差产生错位爽差 | 体系错位/时间错位/空间错位/反套路（上限最高） |
| **模拟推演/情报流** | 零成本试错，未卜先知 | 每条情报有明确目的，主角需主观分析判断 |
| **熟练度面板** | 线性期待感，付出一分收获二分 | 信息差=期待感，制造落差打压后反转（入门最友好） |
| **反套路金手指** | 预期颠覆+反转爽感 | ✅立锚点 → ✅亲手打碎。先建立"理所当然"认知，再颠覆 |

### Step 2：金手指三部分简介

金手指展示遵循三段式：
1. **开篇事件简述** — 最短篇幅交代主角当前处境
2. **金手指展示** — 数据化呈现"能做什么"
3. **反套路核心梗** — 这金手指的与众不同，一句话概括

### Step 3：按类型设计

#### 熟练度流
- 核心：练了就涨进度，单一机制深挖
- 信息差：主角面板显示进度，配角不知情并质疑/打压
- 等待1-5章制造落差再释放，落差越大爽感越强
- 数值呈现：`入门 34/100 → 熟练 67/100 → 精通 99/100`

#### 模拟推演/情报流
- 每条情报归类为：长线事件 / 眼前危机 / 背景穿插
- 情报有真有误，需要主角拼凑解读（提高门槛）
- **必须给实力数值化成长线**，否则后期粘性不足

#### 错位金手指
- 背景下级+金手指上级 → 降维打击（碾压错位）
- 背景上级+金手指下级 → 靠脑洞反杀（逆向错位，更高级）
- 时间/空间错位 → 规则交叉碰撞

#### 词条海克斯
- 表面释义 → 有趣演绎（"坚如磐石"≠防御强，而是"不动"）
- 机制可以花，但实力提升必须清晰可见

#### 反套路金手指
- 第一步：立锚点（前20章让读者认为金手指是X）
- 第二步：亲手打碎（第25章发现金手指其实是Y）
- 打碎必须有前文伏笔，不能机械降神

### Step 4：关联剧情

- 每个事件必须关联金手指，不关联 → 删
- 主角不能是打工机器人（系统发什么做什么）
- **化困境为食粮**：困境（邪祟/毒素/诅咒）→ 金手指能吞噬/转化 → 变成修为
- 困境必须真的有危险，否则没有紧张感

### Step 5：设计代价

| 代价类型 | 说明 | 案例 |
|---------|------|-----|
| 硬代价 | 寿命、资源、风险 | 推演消耗灵石 |
| 软代价 | 时间、冷却次数 | 冷却24小时 |

- 不能太轻（形同虚设）也不能太重（不敢用导致剧情停滞）
- 代价需要解决 → 推动赚资源/找办法的新剧情

### Step 6：小杠杆撬大石头

金手指初始功能必须"小"到不起眼的程度，主角通过洞察力用到极限，产出"原来还能这样用"的惊喜感。

## 自检清单

- [ ] 金手指与每个事件有关联？
- [ ] 只有一个核心机制（没有不断加新功能）？
- [ ] 主角保持主观能动性（不是系统打工机器人）？
- [ ] 有明确的信息差/期待感设计？
- [ ] 使用代价合理？
- [ ] 数值呈现简单直观？
- [ ] 困境有真实危险性？
- [ ] 金手指没有太强太全能（还有能难住主角的事件）？
