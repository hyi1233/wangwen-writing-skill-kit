# 网文开篇设计

> 核心信条：网文不是顺着写的，是倒果为因的。先定后续引爆点，再倒推开篇布局。

## 触发条件

用户提到以下任一场景时自动激活：
- 写开篇 / 写开头 / 写黄金三章 / 写前三章
- 设计书名 / 写简介
- 穿越开局架构设计
- 网文开篇审查

## 核心规则

1. **倒果为因** — 先确定后续要引爆的爽点场景，再逆向倒推开篇该铺垫什么
2. **多重功能** — 每一句话必须承担≥1功能：带设定、刻人设、推剧情、埋伏笔
3. **困境+希望成对出现** — 没有希望的困境=给读者吃屎；没有困境的希望=空洞意淫
4. **90%篇幅遵循** — 设定穿插 → 主观反馈 → 驱动确立
5. **信息3-4章内引爆** — 前置信息不能等到30章后才用上

## 工作流

### Step 1：书名设计

书名必须传达**金手指/题材/核心卖点**中至少一个。

- 读者通过书名判断"这是本什么书"
- 书名 → 简介 → 正文风格必须一致
- 推荐模板：《[金手指]+[行为]》《从×××开始×××》《我×××》

### Step 2：简介三段式（三要素缺一不可）

| 顺序可调 | 内容 | 要点 |
|---------|------|------|
| 开局钩子 | 困境/反常事件 | 颠覆预期，几十到上百字 |
| 核心看点 | 金手指出现 | 让读者感知它能解决开局事件 |
| 全书脉络 | 主角立场/核心梗 | 读者寻找认同的剧情倾向 |

**变体**：困境式 / 反常式 / 反差身份式。同人/历史文只需开篇节点+全书脉络。

### Step 3：开篇六要素（缺一即不成立）

| # | 要素 | 说明 |
|---|------|------|
| 1 | 舞台搭建 | 开篇即给出世界信息 |
| 2 | 事件起因/今生延展性 | 原主遗留的驱动事件，非简单背景介绍 |
| 3 | 合理逻辑目标 | 主角结合现状产生的驱动力 |
| 4 | 明确变量 | 穿越者身份 + 金手指 |
| 5 | 解题行进 | 朝目标推进的具体剧情 |
| 6 | 制造落差 | 有起伏才有爽点爆发 |

**致命错误**：①只堆世界观没有驱动事件；②主角被动等剧情推

### Step 4：睁眼穿架构

两种开局方式选其一：

**A. 直白切入**：两句话 = 时间/地点/天气 + 主角正在什么情境

**B. 分总式穿越确认**：
1. 先感知局部异常（陌生的床/衣服/身体）
2. 整合记忆碎片，确认穿越事实
3. 给出主角主观反馈和目标确立
4. 控制在1-2章内完成

**前世今生衔接**：
- 有用写前世，没用直接写今生
- 最佳衔接：前世死亡/遗憾引导今生
- 前世技能必须在今生有作用

**四种开篇节点选择**：
- 危机节点（最紧凑）
- 转折节点（穿越/觉醒/退婚）
- 日常节点（从日常拉入异常）
- 高潮前置（吸引力最强）

### Step 5：对话开局（如选择对话开局）

- **普适性**：读者必须理解（"副统领""银钱"有通用认知）
- **信息密度**：每句带设定或推剧情
- 禁止"你好""吃了吗"式无效寒暄
- 对话后紧跟主角主观反馈

### Step 6：环境描写

- **要么不做形容词，做了就必须有目的**
- 正衬：环境烘托当下氛围
- 反衬：环境反差制造转折感
- 使用具体可感元素，禁止空洞形容词（"阴森恐怖""美丽动人"）

### Step 7：设定穿插

- **从轮廓到清晰**：大世界观 → 收束到主角个人
- **走哪写到哪**：看到什么写什么，有次序补充
- **一事不烦二主**：出现的角色/设定后续必须有明确用途
- **转回正在进行时**：禁止用"与此同时""三年后"生硬切走

## 自检清单

写完后逐项检查：

- [ ] 开篇六要素齐全？
- [ ] 困境 + 希望成对出现？
- [ ] 简介三要素（困境、金手指、全书脉络）齐全？
- [ ] 每个环境形容词后文有呼应？
- [ ] 每个开局角色/设定后续有用途？
- [ ] 设定穿插遵循"走哪写到哪"？无设定堆叠？
- [ ] 前置信息能在3-4章内引爆？
- [ ] 主角在关键信息后给了主观反馈和驱动目标？
- [ ] 书名传达了金手指/题材/卖点？
- [ ] 书名 → 简介 → 正文风格统一？
